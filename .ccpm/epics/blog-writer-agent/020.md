# T8: Docker and Deployment Configuration

## Context

The blog-writer-agent must be deployed on a VPS via Coolify, which requires Docker containerization. The deployment setup needs to be straightforward: build a Docker image with Python 3.12, install dependencies, expose a single port, and start via Uvicorn. The container must correctly load configuration from `.env` (secrets) and `config.yaml` (app settings), and handle the Google service account JSON file either via mounted volume or injected secret.

## Description

Create a Dockerfile using Python 3.12 slim base image with Uvicorn entrypoint, a docker-compose.yml for local development, and a .dockerignore to exclude unnecessary files. Verify the container builds, runs, exposes the correct port, and loads configuration properly. Handle the service account JSON file via volume mount (local dev) or Coolify secret file injection (production).

## Implementation Steps

1. **Create `Dockerfile`:**
   ```dockerfile
   FROM python:3.12-slim
   WORKDIR /app
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   COPY . .
   EXPOSE 8000
   CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
   ```
   - Use multi-stage build only if image size becomes an issue (unlikely for Python)
   - Do NOT copy `.env` into image — secrets injected at runtime

2. **Create `docker-compose.yml`:**
   ```yaml
   version: "3.8"
   services:
     blog-writer:
       build: .
       ports:
         - "8000:8000"
       env_file:
         - .env
       volumes:
         - ./config.yaml:/app/config.yaml:ro
         - ./service-account.json:/app/service-account.json:ro
       restart: unless-stopped
   ```
   - Volume mount for `config.yaml` and service account JSON
   - `env_file` for `.env` secrets
   - `restart: unless-stopped` for reliability

3. **Create `.dockerignore`:**
   ```
   .git
   .ccpm
   __pycache__
   *.pyc
   .env
   service-account.json
   tests/
   .pytest_cache
   .venv
   *.md
   !requirements.txt
   ```
   - Exclude secrets, tests, cache, and documentation from image

4. **Add deployment notes to `config.yaml.example`:**
   - Comment explaining Coolify environment variable injection
   - Comment explaining service account file path configuration
   - Example `GOOGLE_SERVICE_ACCOUNT_PATH=/app/service-account.json`

## Acceptance Criteria

- **NFR-5 / docker-build:** GIVEN a Dockerfile in the repo, WHEN `docker build -t blog-writer .` is run, THEN the image builds successfully with no errors.
- **NFR-5 / docker-run:** GIVEN the built image, WHEN `docker run` is executed, THEN the container starts Uvicorn on port 8000.
- **NFR-5 / port-exposed:** GIVEN the running container, WHEN a request is sent to port 8000, THEN the FastAPI server responds.
- **NFR-5 / config-loaded:** GIVEN `.env` and `config.yaml` are provided, WHEN the container starts, THEN configuration is loaded correctly.
- **NFR-5 / secrets-not-in-image:** GIVEN the built Docker image, WHEN inspected, THEN `.env` and `service-account.json` are NOT present in any layer.

## Interface Contract

**Receives from 001:**
- `requirements.txt` — Python dependencies
- `app/main.py` — Uvicorn entry point (`app.main:app`)

**Produces:**
- `Dockerfile` — production-ready container definition
- `docker-compose.yml` — local development setup
- `.dockerignore` — build exclusion rules
- Deployable Docker image compatible with Coolify

## Tests to Write

1. **test_docker_build:** Run `docker build -t blog-writer-test .`. Assert exit code 0.
2. **test_docker_run:** Run container, wait 5 seconds, `curl http://localhost:8000/webhook` (expect 401 — no API key). Assert server is responding.
3. **test_secrets_not_in_image:** Run `docker history blog-writer-test` and `docker run blog-writer-test ls -la`. Assert `.env` and `service-account.json` are not present.
4. **test_config_yaml_mount:** Run container with volume-mounted `config.yaml`. Assert server starts without config errors.

Note: Docker tests are integration tests that require Docker daemon. They should be run manually or in CI with Docker available.

## Technical Details

- Python 3.12 slim image is ~150MB base. Final image will be ~300-400MB with dependencies.
- Uvicorn is run without `--reload` in production (Dockerfile). For local dev, docker-compose can override the CMD with `--reload` if needed.
- Service account JSON is the most sensitive file. In Coolify, it can be injected as a secret file or as an environment variable (base64-encoded JSON). The config system should handle both: file path (default) or base64 env var (fallback).
- `.dockerignore` must exclude `*.md` to prevent documentation from inflating image size, but must NOT exclude `requirements.txt` (hence the `!requirements.txt` override).
