# T6: Google Docs Creation and Formatting

## Context

The blog-writer-agent's final output is a publication-ready Google Doc with native formatting — not a raw markdown file. Editors need to receive documents with proper headings (H1-H6 as Google Doc Heading styles), bold, italic, bullet lists, and blockquotes rendered as native formatting. The Google Docs API requires a specific approach: insert all text first, then apply formatting in reverse document order (because insertions shift indices). This task implements both the markdown-to-Docs-API conversion and the Drive API integration for doc creation and sharing. Per plan-review DRY-1, docs and drive operations are merged into a single `google_docs.py` file.

## Description

Implement two components: (1) a markdown parser (`markdown_parser.py`) that converts markdown text into Google Docs API batch request objects, and (2) a Google Docs service (`google_docs.py`) that creates a blank Doc in the configured Drive folder, applies formatted content via Docs API batch update, and sets sharing permission via Drive API. The sharing role is configurable (default "writer" — anyone with link can edit, per AD-11). The markdown parser handles: H1-H6 headings, bold (`**text**`), italic (`*text*`), bullet lists (`- item`), and blockquotes (`> text`).

## Implementation Steps

1. **Create `app/services/markdown_parser.py`:**
   - Implement `parse_markdown(markdown: str) -> tuple[str, list[dict]]`:
     - First pass: extract plain text (strip markdown syntax), track positions of each element
     - For headings: map `# H1` -> `HEADING_1`, `## H2` -> `HEADING_2`, etc. through H6
     - For bold: find `**text**` pairs, record start/end indices in plain text
     - For italic: find `*text*` pairs (not inside bold), record indices
     - For bullet lists: find lines starting with `- `, record as `BULLET_DISC` list items
     - For blockquotes: find lines starting with `> `, record for indent formatting
     - Second pass: build Docs API request objects in REVERSE order (critical for correct index positioning)
     - Return tuple: (plain_text, list_of_batch_requests)

2. **Build Docs API request objects:**
   - Heading: `{"updateParagraphStyle": {"range": {"startIndex": S, "endIndex": E}, "paragraphStyle": {"namedStyleType": "HEADING_N"}, "fields": "namedStyleType"}}`
   - Bold: `{"updateTextStyle": {"range": {"startIndex": S, "endIndex": E}, "textStyle": {"bold": True}, "fields": "bold"}}`
   - Italic: `{"updateTextStyle": {"range": {"startIndex": S, "endIndex": E}, "textStyle": {"italic": True}, "fields": "italic"}}`
   - Bullet list: `{"createParagraphBullets": {"range": {"startIndex": S, "endIndex": E}, "bulletPreset": "BULLET_DISC_CIRCLE_SQUARE"}}`
   - Blockquote: `{"updateParagraphStyle": {"range": {"startIndex": S, "endIndex": E}, "paragraphStyle": {"indentFirstLine": {"magnitude": 36, "unit": "PT"}, "indentStart": {"magnitude": 36, "unit": "PT"}}, "fields": "indentFirstLine,indentStart"}}`

3. **Create `app/services/google_docs.py`:**
   - Import `googleapiclient.discovery`, `google.oauth2.service_account`
   - Implement `GoogleDocsService` class:
     - `__init__(self, settings)`: authenticate via service account, build `drive` and `docs` service objects
     - `async def create_formatted_doc(self, title: str, markdown_content: str) -> str`:
       - Step 1: Create blank Doc in Drive folder via `drive.files().create(body={"name": title, "mimeType": "application/vnd.google-apps.document", "parents": [settings.drive_folder_id]})`
       - Step 2: Parse markdown via `parse_markdown(markdown_content)` -> (plain_text, format_requests)
       - Step 3: Insert plain text via `docs.documents().batchUpdate(documentId=doc_id, body={"requests": [{"insertText": {"location": {"index": 1}, "text": plain_text}}]})`
       - Step 4: Apply formatting via `docs.documents().batchUpdate(documentId=doc_id, body={"requests": format_requests})`
       - Step 5: Set sharing via `drive.permissions().create(fileId=doc_id, body={"role": settings.doc_share_permission, "type": "anyone"})`
       - Return Doc URL: `f"https://docs.google.com/document/d/{doc_id}/edit"`

4. **Handle formatting order:**
   - All formatting requests MUST be applied in reverse document order (highest index first)
   - This prevents earlier formatting from shifting indices of later formatting
   - The `parse_markdown()` function returns requests pre-sorted in reverse order

5. **Error handling:**
   - Wrap each API call in try/except
   - If doc creation fails: raise with clear error
   - If formatting fails: log warning but return doc URL (partially formatted doc is better than no doc)
   - If sharing fails: log warning, return URL (doc exists but is private)

## Acceptance Criteria

- **FR-7 / doc-created-in-folder:** GIVEN a markdown article, WHEN `create_formatted_doc()` is called, THEN a Google Doc is created in the configured Drive folder.
- **FR-7 / headings-formatted:** GIVEN markdown with `## Heading`, WHEN the Doc is created, THEN the heading is styled as `HEADING_2` in Google Docs (not plain text with `##`).
- **FR-7 / bold-italic-formatted:** GIVEN markdown with `**bold**` and `*italic*`, WHEN the Doc is created, THEN text is rendered with native bold and italic formatting.
- **FR-7 / bullet-lists-formatted:** GIVEN markdown with `- item1\n- item2`, WHEN the Doc is created, THEN items are rendered as native Google Doc bullet list items.
- **FR-7 / sharing-permission:** GIVEN `doc_share_permission: "writer"` in config, WHEN the Doc is created, THEN the Doc is shared as "anyone with link can edit".
- **FR-7 / doc-url-returned:** GIVEN a successfully created Doc, WHEN the function returns, THEN it returns the full Google Docs URL.
- **FR-7 / no-raw-markdown:** GIVEN a formatted Doc, WHEN an editor opens it, THEN no raw markdown symbols (*, #, **, _) are visible.

## Technical Details

- Google Docs API uses 1-based indexing. The document body starts at index 1. Text insertion at index 1 pushes existing content forward.
- Formatting requests use `range` with `startIndex` and `endIndex`. These indices refer to positions in the final plain text, NOT the markdown source.
- The reverse-order requirement for batch requests is because the Docs API processes requests sequentially. If you format index 10-20 first, then insert text at index 5, the 10-20 range shifts. Processing in reverse (highest index first) avoids this.
- `google-api-python-client` is synchronous. Wrapping calls in `asyncio.to_thread()` or `loop.run_in_executor()` prevents blocking the event loop.
- Bold/italic detection must handle nested cases: `***bold and italic***` should apply both styles. Edge case: handle gracefully or skip.
- Per plan-review DRY-1, `google_docs.py` combines what was originally planned as separate `docs.py` and `drive.py` files. Both use the same service account credentials.
